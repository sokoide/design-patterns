# はじめに：Go Design Patterns 〜Clean Architecture への旅〜

ようこそ、デザインパターンの世界へ！
この本は、単に「デザインパターン」を暗記するためのものではありません。
**Go 言語** の特性を活かし、**Clean Architecture** の思想を取り入れながら、
「変更に強く、テストしやすいコード」を書くための冒険の書です。

## 🎯 この本の目的

1. **「なぜ」を理解する**: パターンが生まれた背景と、解決する課題を学びます。
2. **Go らしく書く**: Java や C++の直訳ではなく、Go のイディオム（Interface, Struct, Embedding）に沿った実装を習得します。
3. **Clean Architecture を体感する**: ビジネスロジック（Domain）と技術的詳細（Adapter）を切り離す設計を実践します。

## 👥 対象読者

- **Go 言語の基礎**（文法、構造体、インターフェース）を理解しているエンジニア
- 「コードがスパゲッティになってきた…」と悩んでいる中級者
- アーキテクチャ設計に興味があるが、どこから手をつければいいかわからない方

## 🗺️ この本の歩き方

この本は、Go で真に重要となるデザインパターンを効率的に学ぶように構成されています。

| Day       | テーマ                                                                                                    | 対象パターン                                                     | 目標                                                                           |
| :-------- | :-------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------------- | :----------------------------------------------------------------------------- |
| **Day 1** | [**Go らしい初期化** (Initialization)](./01_day1_creational_ja.md)                                        | **Functional Options**                                           | Go で最も多用される柔軟な初期化パターンを学ぶ。                                |
| **Day 2** | [**構造の整理と拡張** (Structural 1)](./02_day2_structural_1_ja.md)                                          | Adapter, Decorator, Composite                                    | 異なるインターフェースの統合や、機能の動的な追加方法を学ぶ。                   |
| **Day 3** | [**複雑さの隠蔽と制御** (Structural 2)](./03_day3_structural_2_ja.md)                                | Facade, Proxy, Flyweight                                         | 複雑なサブシステムの整理、アクセス制御、リソース効率化を学ぶ。                 |
| **Day 4** | [**アルゴリズムと通知** (Behavioral 1)](./04_day4_behavioral_1_ja.md)                               | Strategy, Observer, Command                                      | アルゴリズムの切り替え、状態変化の通知、操作のオブジェクト化を学ぶ。           |
| **Day 5** | [**状態と責任の管理** (Behavioral 2)](./05_day5_behavioral_2_ja.md)                               | Chain of Responsibility, State, Memento                          | 責任の連鎖による処理、状態遷移の管理、状態の保存と復元を学ぶ。                 |

各章（Day）は以下のセクションで構成されています。

1. **ストーリー**: 難しい概念を身近な例え話で直感的に理解します。
2. **コンセプト**: パターンの定義と構造を Mermaid ダイアグラムで可視化します。
3. **Go 実装の極意**: このリポジトリのコードを例に、実装のポイントを解説します。
4. **ハンズオン**: 実際に手を動かして理解を深める小さな課題です。
5. **クイズ**: 理解度を確認するための 3 択クイズです。
