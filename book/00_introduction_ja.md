# はじめに：Go Design Patterns 〜Clean Architecture への旅〜

ようこそ、デザインパターンの世界へ！
この本は、単に「デザインパターン」を暗記するためのものではありません。
**Go 言語** の特性を活かし、**Clean Architecture** の思想を取り入れながら、
「変更に強く、テストしやすいコード」を書くための冒険の書です。

## 🎯 この本の目的

1. **「なぜ」を理解する**: パターンが生まれた背景と、解決する課題を学びます。
2. **Go らしく書く**: Java や C++の直訳ではなく、Go のイディオム（Interface, Struct, Embedding）に沿った実装を習得します。
3. **Clean Architecture を体感する**: ビジネスロジック（Domain）と技術的詳細（Adapter）を切り離す設計を実践します。

## 👥 対象読者

- **Go 言語の基礎**（文法、構造体、インターフェース）を理解しているエンジニア
- 「コードがスパゲッティになってきた…」と悩んでいる中級者
- アーキテクチャ設計に興味があるが、どこから手をつければいいかわからない方

## 🗺️ この本の歩き方

この本は、23 の GoF デザインパターンを **5 日間** で制覇するように構成されています。

| Day       | テーマ                                                                                                    | 対象パターン                                                     | 目標                                                                           |
| :-------- | :-------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------------- | :----------------------------------------------------------------------------- |
| **Day 1** | [**オブジェクト生成の基礎** (Creational)](./01_day1_creational_ja.md)                                        | Singleton, Factory Method, Abstract Factory, Builder, Prototype  | オブジェクトの生成と利用を切り離す方法を学ぶ。                                 |
| **Day 2** | [**構造の整理と拡張** (Structural 1)](./02_day2_structural_1_ja.md)                                          | Adapter, Bridge, Composite, Decorator                            | 異なるインターフェースの統合や、機能の動的な追加方法を学ぶ。                   |
| **Day 3** | [**効率化とインターフェース** (Structural 2 & Behavioral Intro)](./03_day3_structural_2_behavioral_1_ja.md) | Facade, Flyweight, Proxy, Strategy                               | 複雑さの隠蔽、リソース効率化、そしてアルゴリズムの切り替え（Strategy）を学ぶ。 |
| **Day 4** | [**振る舞いの分離と通知** (Behavioral 1)](./04_day4_behavioral_2_ja.md)                                      | Observer, Command, State, Template Method, Iterator              | 状態変化の通知、処理のカプセル化、状態管理を学ぶ。                             |
| **Day 5** | [**複雑なフローと責任** (Behavioral 2)](./05_day5_behavioral_3_ja.md)                                        | Chain of Responsibility, Mediator, Memento, Visitor, Interpreter | 複雑なオブジェクト間の連携や、責任の連鎖、データ構造と処理の分離を学ぶ。       |

各章（Day）は以下のセクションで構成されています。

1. **ストーリー**: 難しい概念を身近な例え話で直感的に理解します。
2. **コンセプト**: パターンの定義と構造を Mermaid ダイアグラムで可視化します。
3. **Go 実装の極意**: このリポジトリのコードを例に、実装のポイントを解説します。
4. **ハンズオン**: 実際に手を動かして理解を深める小さな課題です。
5. **クイズ**: 理解度を確認するための 3 択クイズです。
